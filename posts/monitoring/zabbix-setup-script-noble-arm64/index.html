Zabbix setup script for Ubuntu Noble ARM64/AArch64

    From Wikipedia:

    Zabbix is an open-source software tool to monitor IT infrastructure such as networks, servers, virtual machines, and cloud services.

    This script automate the installation of Zabbix Server, Frontend and Agent along with its dependencies. The script was tested on an LXD container using the Ubuntu 24.04.1 image, running in a VM with the VM.Standard.A1.Flex shape on an Oracle Cloud Infrastructure compute instance. I believe it should work on any machine having the same OS and processor.

    The script

    How to run

    Directly from GitHub Gist

    curl -s https://gist.githubusercontent.com/musahi0128/7252c38afd9dbb0d29e103d05eba263b/raw | sudo bash

    Manually

    1. Copy the script
    2. Create a new file via nano
      nano zabbix-setup-arm64.sh
    3. Paste the script
    4. Save (Ctrl+S) and exit nano (Ctrl+X)
    5. Make the file executable
      chmod a+x zabbix-setup-arm64.sh
    6. Execute the file with sudo
      sudo ./zabbix-setup-arm64.sh

    Continue with Zabbix setup

    Once the installation completed, open the Zabbix frontend and proceed with the setup from there.

    Secure your installation

    Option 1: Change Admin password

    1. Sign in as the Admin user.
    2. On the left sidebar, click on User settings â†’ Profile
    3. Under User tab you can click on Change password.

    Option 2: Setup a new Super admin user and disable Admin user

    1. Setup a new Super admin user
      1. Sign in as the Admin user.
      2. On the left sidebar, click on Users â†’ Users.
      3. On the right-top part of the page, click on Create user.
      4. Fill in all the necessary information on the User tab.
      5. Change to Permissions tab then click on Select button for the Role.
      6. Click on Super admin role.
      7. At the bottom part of the page, click on Add.
    2. Disable Admin user
      1. Sign out from Admin user then sign in as the newly created user.
      2. On the left sidebar, click on Users â†’ Users.
      3. Click on Admin user.
      4. On the User tab, click on Select button for the Groups.
      5. Check the Disabled group then click on Select button.
      6. At the bottom part of the page, click on Update.
