Azure Free Virtual Machine

    Summary

    Azure offer Free services for 12 months for new customer which you can check by following steps:

    1. Go to Subscriptions.
    2. Click on your Subscription name.
    3. Under Top free services by usage section, click on View all free services button.

    With the free offer, you can provision following VM without additional charge:

    • Free account’s virtual machine (Windows)
    • Free account’s virtual machine (Linux)

    You might want to provision the VM manually to have more control. For instance, you want to use the latest Ubuntu Noble image instead of Ubuntu Jammie which you will get by provisioning the VM using the URL above. However, using the default option when manually provisioning the VM will likely result in additional charges.

    To make sure the VM uses the free offers, following details needs to be specified:

    • Basics
      • Availability options: No infrastructure redundancy required
      • Size: Standard_B1s - 1 vcpu, 1 GiB memory ($7.59/month) (free services eligible)
    • Disks
      • OS disk size: 64 GiB (P6)
      • OS disk type: Premium SSD (locally-redundant storage)
    • Networking
      • Public IP: Create new with SKU: Basic and Assignment: Dynamic

    I lost access to my free subscription benefits somehow, and now I’m on Pay-as-you-go subscription. Fortunately, the free offer still remains and those configs help me avoid additional charges.

    Details

    Basic

    Availability options

    I’m not so sure myself about this one, but when selecting options other than No infrastructure redundancy required, I get the impression that multiple VM will be provisioned or may allocate additional resources (e.g., replication for high availability), which exceed the free offer.

    Size

    The free offer listed following size:

    • Virtual Machines, Basv2 Series Windows, B2ats v2
    • Virtual Machines, Basv2 Series, B2ats v2
    • Virtual Machines, Bpsv2 Series Windows, B2pts v2
    • Virtual Machines, Bpsv2 Series, B2pts v2
    • Virtual Machines, BS Series Windows, B1s
    • Virtual Machines, BS Series, B1s

    However:

    • B2ats and B2pts require 2 vCPUs, and the message when trying to select those size are Insufficient quota and Family vCPUs are insufficient to deploy these sizes.
    • When doing provisioning manually, all Windows image required at least 127 GiB on the OS disk size, so it won’t be covered by the free offer.

    So Standard_B1s, which giving you 1 vCPU and 1 GiB memory, is the only option if you want to use the free offer. Unless provisioned using the URL mentioned on the summary of this post, or via Terraform (check out this post), you can only manually provision a Linux VM due to disk size constraints.

    The free offer’s usage/limit was 750 (1 Hour), meaning you can have one VM running a month for free. If you provision two VMs, then both can run for around half of a month before incurring additional charges. And so on and so forth. Please note that this scenario would work and stays within the free offer if you use only one public IP.

    Disk

    Both of OS disk size (64 GiB (P6)) and type (Premium SSD (locally-redundant storage)) need to match exactly, as the meter says Storage, Premium Page Blob, P6 Disks.

    Networking

    Networking, Public IP Addresses, IP Address Hours listed on the free services page entitled you to one public ip with Basic SKU and Dynamic address assignment.

    Using Standard SKU, be it static or dynamic address assignment; or Basic SKU with static address assignment, would still cost you to run the VM. It would even incurring additional charges when the VM is not running, as the address is taken from the pool and allocated indefinitely to your subscription until the public ip is released.

    Usage checking

    Usage on your Free services for 12 months will start to appear the next day. You should check for following meters:

    • Virtual Machines, BS Series, B1s
    • Storage, Standard HDD Managed Disks, Disk Operations
    • Storage, Premium Page Blob, P6 Disks
    • Networking, Public IP Addresses, IP Address Hours

    If you still getting additional charges somehow, you can check which resource are getting charged. To identify the resources:

    1. Go to Subscriptions page on Azure portal.
    2. Click on your Subscription name.
    3. On Spending rate and forecast section, click on View details.
    4. Change the View to CostByResource

    As the view name implied, this will show you cost by resource, and it should be clear which resource are getting charged.
