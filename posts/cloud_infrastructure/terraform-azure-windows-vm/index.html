Provision Azure's free Windows VM using Terraform

    Summary

    My previous post described about how to provision a free VM in Azure with the Free services for 12 months offer and details you need to take note to make sure the resource isn’t getting additional charge.

    On this post, I would like to show how to do the same using Terraform, specifically provision a Windows VM. Linux VM will be discussed on the next post.

    HashiCorp Terraform is an infrastructure as code tool that lets you define both cloud and on-prem resources in human-readable configuration files that you can version, reuse, and share. You can then use a consistent workflow to provision and manage all of your infrastructure throughout its lifecycle. Terraform can manage low-level components like compute, storage, and networking resources, as well as high-level components like DNS entries and SaaS features. 1

    If you are new to Terraform, head over to this page to learn the basic of Terraform with Azure.

    The code

    # main.tf

# terraform.tf

terraform {
  required_providers {
    azurerm = {
      source  = "hashicorp/azurerm"
      version = ">= 4.14.0"
    }
  }
  required_version = ">= 1.10.3"
}

# provider.tf

provider "azurerm" {
  features {
    virtual_machine {
      skip_shutdown_and_force_delete = true
      delete_os_disk_on_deletion     = true
    }
  }
  subscription_id = var.subscription_id
}

# variables.tf

variable "prefix" {
  type    = string
  default = "free-win-vm"
}

variable "location" {
  type    = string
  default = "East US"
}

variable "subscription_id" {
  type = string
}

variable "admin_password" {
  type      = string
  sensitive = true
}

# main.tf

resource "azurerm_resource_group" "main" {
  name     = "${var.prefix}-rg"
  location = var.location
}

resource "azurerm_network_security_group" "main" {
  name                = "${var.prefix}-nsg"
  location            = azurerm_resource_group.main.location
  resource_group_name = azurerm_resource_group.main.name

  security_rule {
    name                       = "Allow-RDP"
    priority                   = 1001
    direction                  = "Inbound"
    access                     = "Allow"
    protocol                   = "Tcp"
    source_port_range          = "*"
    destination_port_range     = "3389"
    source_address_prefix      = "*"
    destination_address_prefix = "*"
  }
}

resource "azurerm_virtual_network" "main" {
  name                = "${var.prefix}-vnet"
  location            = azurerm_resource_group.main.location
  resource_group_name = azurerm_resource_group.main.name
  address_space       = ["10.0.0.0/21"]
}

resource "azurerm_subnet" "main" {
  name                 = "${var.prefix}-subnet"
  resource_group_name  = azurerm_resource_group.main.name
  virtual_network_name = azurerm_virtual_network.main.name
  address_prefixes     = ["10.0.2.0/24"]
}

resource "azurerm_windows_virtual_machine" "main" {
  name                  = "${var.prefix}-1"
  resource_group_name   = azurerm_resource_group.main.name
  location              = azurerm_resource_group.main.location
  size                  = "Standard_B1s"
  admin_username        = "azureuser"
  admin_password        = var.admin_password
  patch_mode            = "AutomaticByPlatform"
  network_interface_ids = [azurerm_network_interface.main.id]
  os_disk {
    caching              = "ReadWrite"
    storage_account_type = "Premium_LRS"
    disk_size_gb         = 64
    name                 = "${var.prefix}-1-os-disk"
  }
  source_image_reference {
    publisher = "MicrosoftWindowsServer"
    offer     = "WindowsServer"
    sku       = "2022-datacenter-azure-edition-core-smalldisk"
    version   = "latest"
  }
}

resource "azurerm_public_ip" "main" {
  name                = "${var.prefix}-public-ip"
  location            = azurerm_resource_group.main.location
  resource_group_name = azurerm_resource_group.main.name
  allocation_method   = "Dynamic"
  sku                 = "Basic"
}

resource "azurerm_network_interface" "main" {
  name                = "${var.prefix}-vnic"
  resource_group_name = azurerm_resource_group.main.name
  location            = azurerm_resource_group.main.location
  ip_configuration {
    name                          = "local"
    subnet_id                     = azurerm_subnet.main.id
    private_ip_address_allocation = "Dynamic"
    public_ip_address_id          = azurerm_public_ip.main.id
  }
}

resource "azurerm_network_interface_security_group_association" "main" {
  network_interface_id      = azurerm_network_interface.main.id
  network_security_group_id = azurerm_network_security_group.main.id
}

# outputs.tf

output "public_ip_address" {
  value = azurerm_public_ip.main.ip_address
}

    Workflow

    This assume you are going to run the code locally.

    1. Install Terraform. Make sure the version is the same or newer than what was specified in the script (4.14.0).

    2. Install Azure CLI. The version also have to be the same or newer than what was specified in the script (1.10.3).

    3. Authenticate Azure CLI.

      • Uses your own account; or
      • Create a service principal and uses it to authenticate the cli (recommended).

    4. Fetch your subscription_id here.

      You can also get it from previous step if you authenticate the cli using your own account.

    5. Write the code.

      You can either write the code above into a single main.tf file, or you can use the filename commented on each section of the code to write to separate files.

    6. Write a variable definitions file.

      # main.tfvars

# prefix         = "<your prefix, uncomment and change the value if you want to use different prefix>"
# location       = "<Azure location, uncomment and change the value if you want to use different location>"
subscription_id = "<your subscription id>"
admin_password  = "<admin password to login to the VM later>"

      This will let Terraform command do plan, apply or destroy without prompting you for the variables.

    7. On your terminal, go to the directory where the files is located.

    8. Run terraform init. This will prepare your working directory with required files.

    9. Run terraform plan -var-file='main.tfvars' to see what action Terraform will took and what resources will be provisioned. Check if the would-be provisioned resource conform to the free offer limitation.

    10. If the output contains no error, run terraform apply -var-file='main.tfvars'. It will ask you for confirmation, answer with yes.

    11. The public_ip_address output would be empty and this is a know issue as the time of writing. You can either check directly on Azure portal or run terraform refresh -var-file='main.tfvars' to get it populated.

    Once all the steps has been completed, you can use your preferred rdp client to connect to the vm.
